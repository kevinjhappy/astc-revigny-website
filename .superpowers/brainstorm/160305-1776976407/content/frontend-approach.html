A — Twig + Alpine.js

Rendu serveur (Symfony/Twig) + Alpine.js pour les micro-interactions (menus, sliders, animations légères). Pas de build complexe, très maintenable.

  • ✅ Stack 100% PHP-friendly
  • ✅ SEO natif (rendu serveur)
  • ✅ Déploiement simple
  • ⚠️ Interactivité limitée vs SPA

B — Symfony API + React/Next.js

Symfony expose une API (REST ou API Platform), React/Next.js gère le front. Séparation totale front/back, composants riches, SSR possible.

  • ✅ UX très riche, animations fluides
  • ✅ Architecture découplée
  • ⚠️ Deux codebases à maintenir
  • ⚠️ Complexité accrue pour un club local

C — Twig + GSAP/Swiper (premium UX)

Rendu Symfony/Twig avec des librairies JS ciblées : GSAP pour les animations au scroll, Swiper pour la galerie photos, AOS pour les entrées d'éléments. UX haut de gamme sans SPA.

  • ✅ Résultat visuel impressionnant
  • ✅ SEO natif conservé
  • ✅ Stack maîtrisée côté PHP
  • ⚠️ Quelques libs JS à apprendre
